# Endatix Azure Deployment (Quickstart)

Use this guide to get Endatix running quickly on Azure with Bicep.

`parameters.json` is the primary source of truth for runtime configuration.  
Use local env files only for Hub build-time values.

## Prerequisites

- Azure CLI (`az`)
- SWA CLI (`swa`) for Static Web App deployments
- .NET 10 SDK
- Node.js 22 + pnpm 10

## Required Before You Run Commands

- Runtime values (Bicep): `resource_prefix`, `environment`, `postgres_admin_username`, `postgres_admin_password`
- Runtime secrets: generated by script into `parameters.local.json`
- Hub build-time values: `hub/.env.deploy` (minimum keys). For full list see `hub/.env.example`

## 5-Step Quickstart (Recommended)

### 1) Start from defaults

Use `oss/samples/deployment-scripts/azure/parameters.json` as baseline.  
Only change what you need (prefix, environment, password, optional runtime settings).

### 2) Generate local secrets overrides

Run the helper script (cross-platform Node):

```bash
node ./generate-quickstart-secrets.mjs
```

This creates:
- `parameters.local.json` (Bicep parameter overrides for secrets)
- `hub/.env.deploy` (minimal Hub build-time values)

Both are local-only files and should not be committed.

### 3) Deploy infrastructure (Bicep)

```bash
az group create --name rg-endatix-eval --location eastus

az deployment group create \
  --resource-group rg-endatix-eval \
  --template-file endatix-azure.template.bicep \
  --parameters parameters.json \
  --parameters parameters.local.json \
  --mode Complete
```

### 4) Build and deploy API

```bash
# from Endatix OSS repo root
cd ./src/Endatix.WebHost
dotnet publish -c Release -o ./publish
cd publish
zip -r ../../api.zip .

az webapp deployment source config-zip \
  --resource-group rg-endatix-eval \
  --name eval-endatix-api \
  --src ../../../api.zip
```

### 5) Build and deploy Hub (SWA default)

```bash
# from Hub repo root
cp .env.deploy .env
pnpm build:standalone

swa deploy \
  --output-location .next/standalone \
  --env production \
  --resource-group rg-endatix-eval \
  --app-name eval-endatix-hub \
  --api-language node \
  --api-version 22
```

Token-based variant (CI parity):

```bash
swa deploy \
  --output-location .next/standalone \
  --deployment-token "$HUB_DEPLOYMENT_TOKEN" \
  --api-language node \
  --api-version 22 \
  --env production
```

## Configuration Ownership (Important)

| Bucket | Source of truth | Where to set |
|---|---|---|
| Runtime infrastructure values | Bicep | `parameters.json` (`hubEnvironmentVariables`, `hubAppSettings`, `apiAppSettings`) |
| Runtime secrets | Local override + Azure runtime settings | `parameters.local.json` (generated), then app settings |
| Build-time Hub values | Local env file only | `hub/.env.deploy` before `pnpm build:standalone` |

Notes:
- URLs can be set in Bicep via `hubEnvironmentVariables` and `apiAppSettings`.
- Connection strings are template-managed (`DefaultConnection`, `StorageConnection`).
- Keep `hubRepositoryUrl` and `apiRepositoryUrl` empty for manual/local deployment.

## Alternative: Hub on Web App

If you choose `hubDeploymentMode: "web-app"`:

```bash
pnpm build:standalone
( cd .next/standalone && zip -r ../hub-standalone.zip . )

az webapp config set \
  --resource-group rg-endatix-eval \
  --name eval-endatix-hub \
  --startup-file "node server.js"

az webapp deploy \
  --resource-group rg-endatix-eval \
  --name eval-endatix-hub \
  --src-path hub-standalone.zip \
  --type zip
```

## Developer Loop

1. Update `parameters.json` / `parameters.local.json` and `hub/.env.deploy`
2. Rebuild API and Hub
3. Redeploy API and Hub
4. Verify:
   - `az webapp log tail --resource-group rg-endatix-eval --name eval-endatix-api`
   - `az staticwebapp environment list --name eval-endatix-hub --resource-group rg-endatix-eval -o table`

## Quick Troubleshooting

- SWA warmup timeout: use `--output-location .next/standalone --api-language node --api-version 22`
- Hub can open but API fails: check `NEXT_PUBLIC_API_URL` / `ENDATIX_BASE_URL` and API CORS
- API starts but auth fails: verify generated secrets were applied and redeploy API
